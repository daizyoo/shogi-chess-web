# Release Notes Directory

このディレクトリには、各バージョンのリリースノートが含まれています。

## 命名規則

`RELEASE_NOTES_v{major}.{minor}.{patch}.md`

例: `RELEASE_NOTES_v0.4.0.md`

## リリースノートの作成

新しいリリースを作成する際は、以下の内容を含めてください：

- **リリース日**
- **新機能**: 追加された機能の説明
- **バグ修正**: 修正されたバグの詳細
- **技術詳細**: 実装の詳細や技術的な変更
- **アップグレード方法**: バージョンアップの手順
- **Known Issues**: 既知の問題（もしあれば）
